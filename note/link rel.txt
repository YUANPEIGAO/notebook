<link> 标签常用 rel 属性完整总结（文本格式 + 含用法 + 实例）
一、核心说明
<link> 标签必须写 rel 属性，作用是告诉浏览器当前引入的文件是什么用途，除了 stylesheet 还有大量高频实用值，全部放在 <head> 内使用。
二、最常用 rel 属性（含作用 + 实例）
1. stylesheet（最常用）
作用：引入外部 CSS 样式表实例：<link rel="stylesheet" href="style.css">
2. icon
作用：设置网页标签栏小图标（favicon）实例：<link rel="icon" href="favicon.ico">
3. shortcut icon
作用：兼容旧浏览器的网页图标（和 icon 通用）实例：<link rel="shortcut icon" href="logo.png">
4. preconnect
作用：提前连接第三方域名，加速资源加载（优化性能）实例：<link rel="preconnect" href="https://fonts.googleapis.com">
5. preload
作用：预加载关键资源（字体、图片、JS），优先下载实例：<link rel="preload" href="font.woff2" as="font">
6. prefetch
作用：预判加载（用户可能下一步用到的资源，后台悄悄加载）实例：<link rel="prefetch" href="next-page.html">
7. canonical
作用：声明网页标准链接，解决重复页面 SEO 问题实例：<link rel="canonical" href="https://xxx.com/index.html">
8. alternate
作用：定义替代版本（移动端、其他语言、RSS）实例：<link rel="alternate" hreflang="zh" href="mobile.html">
9. author
作用：指向作者信息实例：<link rel="author" href="humans.txt">
10. license
作用：指向开源协议 / 版权说明实例：<link rel="license" href="LICENSE.html">
三、<link> 标签其他常用属性（不只是 rel）
href作用：资源路径（必填）写法：相对路径、绝对路径、URL
type作用：声明文件类型（现在可省略，浏览器自动识别）实例：type="text/css"、type="image/x-icon"
as配合 preload 使用，声明资源类型可选值：script、style、font、image
media作用：根据设备条件加载样式实例：media="(max-width:768px)"
四、高频组合实战模板（直接复制）
html
预览
<head>
  <!-- 网页图标 -->
  <link rel="icon" href="favicon.ico">
  
  <!-- 引入CSS -->
  <link rel="stylesheet" href="./static/css/style.css">
  
  <!-- 预连接加速 -->
  <link rel="preconnect" href="https://cdn.bootcdn.net">
  
  <!-- 预加载字体 -->
  <link rel="preload" href="font.woff2" as="font">
  
  <!-- SEO标准链接 -->
  <link rel="canonical" href="https://xxx.com">
</head>
五、超简记忆口诀
样式表 = stylesheet
小图标 = icon
加速连接 = preconnect
提前加载 = preload
预判加载 = prefetch
SEO 标准 = canonical
总结
rel 决定 <link> 是干什么用的，不能省略
日常开发最常用：stylesheet、icon、preconnect、preload、canonical
所有 <link> 都必须写在 <head> 里
配合 href、as、media 可实现性能优化 + SEO + 适配
